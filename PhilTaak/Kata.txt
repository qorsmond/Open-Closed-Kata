Ek het n taak vir jou om basic OO SOLID te leer.

Bou n library waar ek 2 verskillende metodes het van serialization. Een wat JSON to serialize en een wat XML to serialize.

Ek daar moet n class wees, serializer, wat ek met 4 class instances kan roep:

Settings
Config
Data
Content

Die class moet dan die instances vat en Settings na n file toe save as XML.
Config as XML.
Data as JSON
Content as JSON.

Die file moet dieselfde naam he as die class name wat vir hom gegee is en kan enige plek wees.

Settings,Config,Data en Content mag net properties op hulle hê, soos n model. Geen logika nie.

Jy mag geen linq, if statements of switch statements gebruik nie.

Nadat jy klaar is, moet jy n nuwe class kan by sit, BinaryContent wat na binary moet serialize. Maar jy moet net classes en methods kan bysit, die huidige code mag nie verander word nie.



____________________________________________________________________________________________________________________________________________________



SOLID Serializer:

Build a library that has 2 different methods of serialization. It should be able to serialize to JSON and XML.

It should have a class (Serializer) with a method (Serialize) that I can call with for different instances of POCO classes:
•	Settings,
•	Config,
•	Data and
•	Content

The POCO class should only have properties, no logic.
The function should take the POCO classes and serialize them to a file that has the same name as the POCO class.

•	Settings should serialize to XML
•	Config should serialize to XML
•	Data to JSON
•	Content to JSON

No conditional if, switch or LINQ statements should be used.

After you are done, you should be able to add a new serialization type, Binary. The POCO class for this type should be “BinaryContent”. The catch is you are only allowed to add new classes and add the IOC binding. NO EXISTING LOGIC SHOULD CHANGE.
